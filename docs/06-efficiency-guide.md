# 효율적인 개발을 위한 가이드

이 프로젝트에서 배운 교훈과 다음 프로젝트를 위한 개선 사항입니다.

## 프로젝트 경험에서 배운 것

### 잘된 점

1. **TypeScript 사용**
   - 컴파일 타임에 타입 에러 발견
   - IDE 자동완성으로 생산성 향상
   - 데이터 구조 명확화

2. **JSON 기반 데이터 관리**
   - 코드와 콘텐츠 분리
   - 비개발자도 데이터 수정 가능
   - 버전 관리 용이

3. **Next.js Image 컴포넌트**
   - 자동 이미지 최적화
   - 반응형 이미지 쉽게 구현
   - 성능 향상

4. **Vercel 자동 배포**
   - Git push만으로 배포
   - 빌드 로그 즉시 확인
   - 롤백 용이

### 개선할 점

1. **이미지 파일명 정규화**
   - 처음부터 통일된 명명 규칙
   - 특수문자, 공백 최소화
   - 확장자 통일

2. **데이터 수집 완료 후 개발**
   - 모든 메타데이터 사전 확보
   - 중간 변경 최소화
   - 명확한 요구사항

3. **프로토타입 먼저**
   - 디자인 확정 후 개발
   - 클라이언트 승인 프로세스
   - UI 변경 비용 절감

## 효율화 전략

### 1. 데이터 파이프라인 자동화

```bash
# 원클릭 데이터 처리
npm run pipeline

# package.json
{
  "scripts": {
    "pipeline": "node scripts/extract-dimensions.js && node scripts/validate-artworks.js"
  }
}
```

### 2. 템플릿 재사용

```
/project-template
├── CLAUDE.md
├── .claude/commands/
├── scripts/
├── docs/
└── src/types/
```

새 프로젝트 시작 시 템플릿 복사로 시간 절약

### 3. 체크리스트 활용

각 단계별 체크리스트로:
- 누락 방지
- 품질 보장
- 일관성 유지

### 4. 문서화 선행

개발하면서가 아닌, 개발 전에:
- 데이터 구조 설계
- API 인터페이스 정의
- 컴포넌트 책임 명확화

## 시간 절약 팁

### 반복 작업 자동화

| 작업 | 수동 | 자동화 |
|------|------|--------|
| 이미지 크기 추출 | 30분 | 1분 |
| 데이터 검증 | 20분 | 10초 |
| 배포 체크 | 15분 | 5분 |

### 재사용 가능한 컴포넌트

```tsx
// 범용 이미지 카드
<ArtworkCard
  data={artwork}
  viewMode="grid|proportional|large"
  onClick={handler}
/>
```

### 스크립트 라이브러리

```bash
scripts/
├── extract-dimensions.js    # 모든 프로젝트에서 사용
├── validate-artworks.js     # 커스터마이즈 가능
├── update-sizes.js          # 템플릿
└── generate-json.js         # 스프레드시트 → JSON
```
